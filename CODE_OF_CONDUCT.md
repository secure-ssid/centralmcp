# Code of conduct

centralmcp welcomes contributors, testers, and users who want better
HPE Networking MCP workflows. Keep collaboration professional, practical, and
safe for people sharing network automation ideas.

## Expected behavior

- Be respectful and constructive in issues, pull requests, reviews, and docs.
- Assume good intent, but accept corrections quickly.
- Keep examples fake or redacted; do not share real credentials, tokens, tenant
  IDs, customer data, private URLs, or sensitive screenshots.
- Focus feedback on the technical change, user impact, validation, and safety
  model.

## Unacceptable behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing another person's private information or customer/network details.
- Posting exploitable security details or real secrets in public issues, pull
  requests, comments, logs, or screenshots.
- Repeatedly derailing discussions away from the project goals.

## Reporting concerns

For security vulnerabilities or credential exposure, follow
[SECURITY.md](SECURITY.md). For other conduct concerns, contact a maintainer
through a GitHub issue or pull request comment using only non-sensitive details.

Maintainers may edit or remove comments, close discussions, or limit
participation when needed to protect contributors and project users.
